Variable Scope - (Var, Let, Const)

Simply, a Scope is an area of execution - where a piece of code is executed.

    Generally, there are 3 types of Scope :
  • Global Scope - variables declared within the global context
  • Functional Scope - variables declared within a functional
  • Block Scope - variables declared with the curly braces '{}'

Var

Variables declared with 'var' keyword have function or global scope depending on where they are declared. As they are not block-level scoped, they are accessible and mutable (liable to change) from outside of the block in which they are declared. This can lead to some unexpected behaviors.

Let & Const

Variables declared with both 'Let' and 'Const' keyword are block scope, meaning they are only accessible within the scope and not from outside. One key difference between the two is value assigned with Let keyword is mutable, whereas Const is not. Although, variables declared with Const, such as non-primitive data types (array, object etc.) can still be modified but can not be reassigned.

Use cases of null and undefined

null

In javaScript the null keyword represents a falsy data type. This means that during conditional statement executions null coercion to false. Null is a primitive data type and represents a object. Although, initially it was a mistake by the creators of javaScript language but fixing it now may cause many web-sites to crash.

    Some common use cases of null
  • Initialization - null is used to initialize variables before assigning a specific value.
  • Default Value - in cases where variable may or may not have a value, it can serve as a default value
  • Error Handling - function or methods may return null to indicate an error in code
  • Testing and Debugging - intentionally setting value to null for testing purposes

undefined

In javaScript, undefined represents the absence of a value or the absence of a variable assignment. It is a primitive-data type.

    Common cases of undefined
  • Variable declared but value not assigned
  • Function parameter is not passed
  • Trying to access missing object properties
  • Function that doesn't have a return statement
  • Accessing an array with an index that doesn't exist
  • Explicitly checking a variable for undefined

REST API

REST (Representational State Transfer) and API (Application Programming Interface) is a set of rules for building and interacting with web services. It is based on principles of REST, that uses standard HTTP/HTTPs methods (GET, POST, PUT, DELETE, PATCH) to perform CRUD (Create, Read, Update, Delete) operations on resources, and representing resources as URLs (Universal Resource Locator).

When a client makes a request for resources through REST API (also known as RESTful API), it transfers a representation a representation of the state of the resource to the client. This information or representation is delivered via one of the several HTTP formats. These includes: JSON, HTML, XLT, Python, PHP, or plaintext. JSON (JavaScript Object Notation) being the popular due to its readability by both humans and machines.

    HTTP/HTTPs Methods
  • GET- Retrieves data from server
  • POST - Create new resource on server
  • PUT - Updates existing resource on server
  • DELETE - Removes an existing resource from server
  • PATCH - If resource exist on server then updates it or if doesn't exist then creates it
    Common HTTP Status Codes
  • 200 (OK)
  • 201 (Created)
  • 204 (No Content)
  • 400 (Bad Request)
  • 404 (Not Found)
  • 500 (Internal Server Error)
